Ensemble¶

Meta algoritmos que combinam vrias tcnicas de aprendizado de mquina em um nico modelo preditivo

Possibilita obter um desempenho preditivo melhor do que o obtido por um modelo nico

Objetivo: melhorar a predio de um modelo base (classificador ou regressor)

Ensemble¶

Ensemble - Tipos de Métodos¶

Os mtodos de ensemble podem ser divididos em dois grupos principais:

  • Sequencial
  • Paralelo

Ensemble - Métodos Sequenciais¶

Modelos base so gerados sequencialmente

Explora a DEPENDNCIA entre os modelos base

O desempenho global pode ser melhorado atribuindo pesos maiores a exemplos classificados erroneamente pelo modelo anterior

Ensemble - Métodos Paralelos¶

Modelos base so gerados em paralelo

Explorar a INDEPENDNCIA entre os modelos base

Erros podem ser reduzidos drasticamente atravs da mdia

Ensemble - Homogêneo x Heterogêneo¶

Homogneo: A maioria dos mtodos de ensemble utilizam um algoritmo base de aprendizado para produzir modelos do mesmo tipo

Heterogneo: Existem alguns mtodos que utilizam algoritmos heterogneos (de diferentes tipos)

Para que os mtodos baseados em ensemble apresentem maior acurcia do que qualquer um de seus membros individuais, os modelos de base devem ser o mais precisos e DIVERSOS possveis

Ensemble - Bagging¶

Bagging: Boostrap Aggregation

Reduo da varincia nas estimativas atravs da agregao (mdia) de mltiplas estimativas

Treina algoritmos em diferentes subconjuntos de dados

As bases de treinamento para cada modelo base so geradas atravs de amostragem aleatria com reposio

Para agregar as sadas dos modelos base, bagging utiliza o voto para classificao e mdia para regresso

Ensemble - Bagging¶

$$ f(x) = \frac{1}{M}\sum_{m=1}^{M}{f_{m}(x)} $$

Random Forests¶

Um exemplo bem comum de classe de algoritmo de ensemble so as florestas aleatrias de rvores (Random Forests)

Em Random Forests, cada rvore um ensemble construdo a partir de uma amostra com substituio.

Alm disso, ao invs de utilizar todas as caractersticas, um conjunto aleatrio de caractersticas selecionado tornando o processo de criao de rvores ainda mais aleatrio.

Como resultado, o vis da floresta cresce ligeiramente mas a varincia diminui significativamente resultando em um modelo muito melhor.

Ensemble - Boosting¶

Famlia de algoritmos que convertem classificadores fracos em algoritmos de aprendizado fortes.

Princpio fundamental: treinar uma sequncia de classificadores fracos que so apenas um pouco melhores que adivinhar aleatoriamente (ex: pequenas rvores de deciso).

Ensemble - Boosting¶

Os classificadores fracos so treinados em verses com peso ponderado dos dados. Mais peso atribudo aos exemplos que foram classificadores errneamente pelo modelo anterior.

As previses so combinadas usando voto ponderado (classificao) ou soma ponderada (regresso) para produzir a estimativa final.

Boosting - AdaBoost¶

Os classificadores fracos no AdaBoost so rvores de deciso com profundida igual a 1, chamados de stumps (tocos).

O AdaBoost pondera as observaes, colocando mais peso na dificuldade de classificar exemplos e menos naqueles que j so classificados corretamente.

Novos classificadores fracos so adicionados sequencialmente e focam seu treinamento nos padres mais difceis.

Exemplos que so difceis de serem classificados recebem cada vez mais peso at que o algoritmo os classifiquem corretamente.

Boosting: AdaBoost¶

  1. Atribua pesos uniformes ao vetor $w$ onde $ \sum_{i} {w_i} = 1 $
  2. Para cada $j$ em $m$ rodadas de boosting, faa:
  3. Treine um classificador fraco: $c_j = train(\mathbf{X}, \mathbf{y}, \mathbf{w})$
  4. Estime as novas classes: $\mathbf{\widehat{y}} = predict(C_j, \mathbf{X})$
  5. Calcule a taxa de erro ponderado: $ \epsilon = \mathbf{w} \cdot (\mathbf{\widehat{y}} == \mathbf{y})$
  6. Calcule o coeficiente: $\alpha_{j} = 0.5 log \frac{1 - \epsilon}{\epsilon}$
  7. Atualize os pesos: $ \mathbf{w} = \mathbf{w} \times exp(-\alpha_j \times \mathbf{\widehat{y}} \times \mathbf{y}) $
  8. Normalize os pesos para que a soma seja 1: $ \mathbf{w} := \mathbf{w} / \sum_i {w_i}$
  9. Realize as estimativas finais: $ \mathbf{\widehat{y}} = (\sum_{j=1}^{m}{(\alpha_j \times predict(C_j, \mathbf{X}})) > 0)$

Onde: $ (\mathbf{\hat{y}} == \mathbf{y}) $ no passo 5 se refere a vetores contendo 0s ou 1s, onde 1 atribudo para estimativas corretas e 0 para erradas

Bagging vs Boosting: Treinamento¶

Bagging vs Boosting: qual é o melhor?¶

No h um vencedor: depende dos dados, da complexidade do problema e do algoritmo base

Bagging e Boosting reduzem a varincia de uma estimativa nica. Modelo com maior estabilidade.

Bagging raramente ir melhorar o vis de um modelo nico com baixo desempenho. Entretanto pode gerar um modelo combinado com menores erros.

Se o problema do modelo nico o overfitting, ento Bagging a melhor opo. Boosting no ajuda a evitar/melhorar o overfitting.

Stacking¶

Stacking uma tcnica de aprendizado por ensemble que combina mltiplos modelos de classificao ou regresso atravs de um meta classificador ou meta regressor.

Os modelos de base so treinados em um conjunto completo da base de treinamento, e ento o meta modelo utiliza as sadas dos modelos de base como caractersticas de entrada.

A base geralmente utiliza diferentes algoritmos de aprendizado e portanto geralmente stacking se refere a um modelo heterogneos de ensemble
